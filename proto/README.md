# TEA Protobuf Definitions

This directory contains the Protocol Buffer (protobuf) definitions for the Transparency Exchange API (TEA) specification.

## Overview

The TEA specification defines a standard API for exchanging product transparency artifacts (SBOMs, VEX, attestations, etc.) between systems. This protobuf implementation provides:

- **Type-safe contracts** - Machine-readable API definitions
- **gRPC services** - High-performance RPC with streaming support
- **REST mapping** - gRPC-Gateway annotations for HTTP/JSON APIs
- **Language bindings** - Buf-managed Go bindings plus Rust bindings generated by `tea-server/build.rs`

## Services

### DiscoveryService (`discovery.proto`)
TEI (Transparency Exchange Identifier) resolution and API discovery.

| RPC | HTTP | Description |
|-----|------|-------------|
| `Discover` | `GET /v1/discovery?tei=...` | Resolve TEI to ProductRelease |
| `GetWellKnown` | `GET /.well-known/tea` | Discovery document |
| `HealthCheck` | `GET /v1/health` | Service health |
| `GetServerInfo` | `GET /v1/info` | Server metadata |

### ConsumerService (`consumer.proto`)
Read-only access to transparency artifacts.

| RPC | HTTP | Description |
|-----|------|-------------|
| `ListProducts` | `GET /v1/products` | List products |
| `GetProduct` | `GET /v1/products/{uuid}` | Get product |
| `ListProductReleases` | `GET /v1/products/{uuid}/releases` | List releases |
| `GetProductRelease` | `GET /v1/product-releases/{uuid}` | Get release |
| `ListComponents` | `GET /v1/components` | List components |
| `GetComponent` | `GET /v1/components/{uuid}` | Get component |
| `GetCollection` | `GET /v1/collections/{uuid}` | Get collection |
| `GetArtifact` | `GET /v1/artifacts/{uuid}` | Get artifact |
| `StreamArtifactContent` | `GET /v1/artifacts/{uuid}/stream` | Download artifact |
| `SearchByIdentifier` | `GET /v1/search/identifier` | Search by ID |
| `SearchByChecksum` | `GET /v1/search/checksum` | Search by hash |

### PublisherService (`publisher.proto`)
Write operations for artifact publishers.

| RPC | HTTP | Description |
|-----|------|-------------|
| `CreateProduct` | `POST /v1/publisher/products` | Create product |
| `UpdateProduct` | `PUT /v1/publisher/products/{uuid}` | Update product |
| `DeleteProduct` | `DELETE /v1/publisher/products/{uuid}` | Delete product |
| `UploadArtifact` | `POST /v1/publisher/artifacts` | Upload artifact |
| `CreateArtifactFromUrl` | `POST /v1/publisher/artifacts/from-url` | Register an artifact from a verified URL |
| `CreateCollection` | `POST /v1/publisher/collections` | Create collection |
| `UpdateCollection` | `POST /v1/publisher/collections/{uuid}/versions` | Create next collection version |
| `SignCollection` | `POST /v1/publisher/collections/{uuid}/sign` | Sign collection |

## Spec vs Reference Server

The protobuf definitions are the canonical TEA contract.

- The `.proto` files define the normative RPC surface, message shapes, and
  lifecycle semantics.
- Reference servers are allowed to expose only a safe subset of the publisher
  API while the remaining operations are still being proven out.
- Unsupported publisher RPCs should fail explicitly, typically with
  `UNIMPLEMENTED`, instead of returning partial success.

### Publisher Reference Profile

The current Rust reference server is intended to be an executable reference for
the stable publisher flows, not a claim that every optional publisher capability
is production-complete.

| RPC group | Canonical contract | Rust reference server |
|-----------|--------------------|-----------------------|
| Product lifecycle | Required when PublisherService is exposed | Implemented |
| Product release lifecycle | Required when PublisherService is exposed | Implemented |
| Component lifecycle | Required when PublisherService is exposed | Implemented |
| Component release lifecycle | Required when PublisherService is exposed | Implemented |
| Artifact upload streaming | Optional advanced publisher capability | Intentionally unimplemented |
| Artifact registration from URL | Optional publisher capability | Implemented with checksum verification and source URL safety checks |
| Artifact delete | Optional publisher capability | Implemented with collection safety checks |
| Collection create/versioned update | Optional publisher capability | Implemented |
| Collection signing | Optional publisher capability | Intentionally unimplemented |
| Batch artifact upload | Optional advanced publisher capability | Intentionally unimplemented |
| Bulk collection import | Optional advanced publisher capability | Intentionally unimplemented |

Reference server note:

- the protobuf surface is canonical and complete
- the current Rust reference server implements discovery / consumer reads fully
- the current Rust reference server implements a safe subset of publisher RPCs today:
  - product create / update / delete
  - product release create / update / delete
  - component create / update / delete
  - component release create / update / delete
  - artifact registration from a verified immutable URL
  - collection create plus versioned update
  - artifact delete with collection safety checks
- the remaining publisher RPCs fail closed with `UNIMPLEMENTED` until their backing storage and versioning semantics are ready

## Generated Code

After running `buf generate`, Buf-managed code is output to `gen/`:

```
gen/
└── go/          # Go protobuf messages and gRPC stubs
```

Rust bindings are generated at Cargo build time into `OUT_DIR` by
`tea-server/build.rs` and exposed through `tea_server::gen`.

`make export-deps` exports Buf registry dependencies into `proto/.buf-deps/` for
IDE support and Rust code generation. The directory is excluded from the Buf
module so `buf lint` and `buf generate` keep working after export.

## Usage Examples

### Go Client

```go
package main

import (
    "context"
    teav1 "github.com/CycloneDX/transparency-exchange-api/gen/go/tea/v1"
    "google.golang.org/grpc"
    "google.golang.org/grpc/credentials/insecure"
)

func main() {
    conn, _ := grpc.Dial("localhost:50051", grpc.WithTransportCredentials(insecure.NewCredentials()))
    defer conn.Close()

    client := teav1.NewConsumerServiceClient(conn)
    
    // Resolve a TEI
    discovery := teav1.NewDiscoveryServiceClient(conn)
    resp, _ := discovery.Discover(context.Background(), &teav1.DiscoverRequest{
        Tei: "urn:tei:uuid:example.com:d4d9f54a-abcf-11ee-ac79-1a52914d44b1",
    })
    
    // Get the product release
    release, _ := client.GetProductRelease(context.Background(), &teav1.GetProductReleaseRequest{
        Uuid: resp.ProductReleaseUuid,
    })
    
    fmt.Printf("Found release: %s v%s\n", release.Uuid, release.Version)
}
```

### Rust Client

```rust
use tea_server::gen::tea::v1::{
    consumer_service_client::ConsumerServiceClient,
    discovery_service_client::DiscoveryServiceClient,
    DiscoverRequest, GetProductReleaseRequest,
};
use tonic::transport::Channel;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let channel = Channel::from_static("http://localhost:50051")
        .connect()
        .await?;

    let mut discovery = DiscoveryServiceClient::new(channel.clone());
    let mut consumer = ConsumerServiceClient::new(channel);

    // Resolve a TEI
    let resp = discovery
        .discover(DiscoverRequest {
            tei: "urn:tei:uuid:example.com:d4d9f54a-abcf-11ee-ac79-1a52914d44b1".into(),
        })
        .await?
        .into_inner();

    // Get the product release
    let release = consumer
        .get_product_release(GetProductReleaseRequest {
            uuid: resp.product_release_uuid,
        })
        .await?
        .into_inner();

    println!("Found release: {} v{}", release.uuid, release.version);
    Ok(())
}
```

## Versioning

This protobuf package follows [Semantic Versioning 2.0](https://semver.org/):

- **MAJOR** version for incompatible API changes
- **MINOR** version for backwards-compatible functionality
- **PATCH** version for backwards-compatible bug fixes

Breaking changes are detected automatically using `buf breaking`.
