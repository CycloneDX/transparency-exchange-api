# Architecture

## Domain-Driven Design

- Bounded contexts
- Ubiquitous language
- Aggregates
- Domain services

## Clean Architecture

- Layers: Domain → Application → Infrastructure → Interface

## Hexagonal Architecture

- Ports & Adapters
- Technology-agnostic domain

## Three Spaces + Proof Space

- Kernel Space: core domain logic, state machines, invariants
- Validation Space: policies, governance, orchestration, checks, promotion gates
- Presentation Space: APIs, UIs, CLIs
- Proof/Governance Space: evidence, attestations, audit, compliance
